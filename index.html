Example: K-means

Simple clustering algorithm

Group similar points together. No labels.

Stuart Lloyd (1957) & E Forgy (1965)

Let's see how a simple ML algorithm work.

Developed independently by the two scientists.

Simple and efficient. May not fit advanced real world applications.

We want to cluster points into 3 clusters.

Random cluster center initialization.

Number of clusters is algorithm parameter. Depends on need and application.

Assign points to nearest cluster center point.

Point color denotes assigned cluster.

Find center (mean) of points in the same cluster.

Update cluster centers.

Repeat: assign every point to nearest cluster center.

Calculate new cluster means.

Update cluster centers.

Repeat...

Repeat. Moving slower now.

Repeat...

Repeat...

Stop when cluster center points stop moving.

Replace coordinates with any other set of numerical features. Age, income, height, credit score, ...

Replace Euclidean distance with any other distance measure.

This algorithm did not need annotated data to learn from. Unsupervised learning.

Targeted advertising

Kroger's 70+% redemption on customer specific coupons.

Kroger (and others) create custom in-mail coupons offers at the customer level.

Narrow tasks it's designed for
  • Identifying dog breeds in photos
  • Targeted advertising
  • Board games

Machine learning is good at narrow tasks it was designed and trained to do.

It's not artificial general intelligence.

What isn't machine learning good at yet?

General intelligence tasks
  • Generalization of problem solving
  • Acting on input not seen before
  • Reasoning

Machine learning models need to train on large data sets before it's put to work.

Cannot act well in new situations.

Machines cannot reason like humans.
